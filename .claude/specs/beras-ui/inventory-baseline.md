## Disposition vocabulary

Only PRD-approved values may enter the machine ledger:

- `implemented`;
- `canonicalized into <Beras export>`;
- `non-visual business boundary` for logic only, while visual artifacts map separately;
- `not ported: dead CSS or known defect`, with evidence;
- `inventory-only: void/crimson`.

`deferred` is forbidden. In tables below, “boundary” text names runtime coupling removed from Beras; disposition after `→` names the visual result.

## Production-source ledger

### Reading key

- Reachability: `R` = statically reachable, `N` = Next route/layout/metadata entry, `U` = zero incoming static imports, `L` = legacy/superseded. Next entries are not called unused merely because imports are absent.
- Current direct visual dependencies: `Ant` = `antd`, `AntI` = `@ant-design/icons`, `RC` = Recharts, `FM` = Framer Motion, `Luci` = Lucide, `Three` = R3F/Drei/react-spring/Three, `TT` = TanStack Table, `RR` = react-responsive, `Native` = React/Next/native CSS only. Business/runtime dependencies are named after `boundary:`.
- Case notation `base [a|b]` expands to stable IDs `base/a`, `base/b`. All case bases have exactly two segments and all expanded IDs satisfy the three-segment catalog contract.
- Responsive/a11y codes: `Shell` = 320 drawer/grid/no page overflow/landmarks; `Form` = labels, error, submit, narrow stack; `Overlay` = focus containment/return, Escape, 320 overflow; `Select` = complete keyboard/long option/name; `Dense` = labelled focusable owned overflow; `Chart` = responsive SVG, data equivalent, marker/non-color meaning; `Cal` = calendar grid/boundary/keyboard; `Tree` = roving tree keyboard/levels; `State` = live region/retry/reduced motion; `Text` = heading/link/safe overflow; `Icon` = decorative hidden/informative named; `Motion` = reduced-motion static equivalent.
- Evidence seed `Snn` means `git show 79927540a3c27d2c29b42d84c42b7e9abcb51800:<source path>` plus the recorded audit reading. `E(base)` means the implementation must attach the visual/responsive/interaction evidence matrix for every expanded case ID under that base. These seeds become explicit `evidenceIds` in the machine ledger; they are not claims that Phase 1 browser evidence already exists.

## Preliminary canonicalization conclusions

- Similar tables, audit logs, metrics, selects, dialogs, states, and export buttons collapse many-to-one; source traceability remains in all 82 rows.
- Active `LeaveCalendarSimple` supplies intent for `LeaveScheduleGrid`; legacy TanStack `LeaveCalendar` remains mapped but its implementation API is discarded.
- `MemberTaskModal` raw `dangerouslySetInnerHTML` path is an unsafe non-portable boundary. Beras renders typed ADF/plain safe React content only; any Phase 2 legacy HTML preservation requires sanitation/conversion in Tere before props reach Beras.
- Sign-in navy hero is an evidenced component variant inside the light product, not evidence for a global dark/void theme.
- Native complex widgets must be live: labels/screenshots alone cannot satisfy chart/table/calendar/tree/dialog/select/loading/`Stat3DScene` mappings.
- Final implementation may refine artifact wording but cannot remove a path, silently merge a distinct state, add `deferred`, or change the baseline without explicit scope reconciliation.
